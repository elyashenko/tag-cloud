# PRD: Live Tag Cloud Voting v1.1

> Интерактивное облако тегов с онлайн-голосованием в реальном времени  
> Версия 1.1 · Апрель 2026 · Статус: Active

## 1. Обзор и цель продукта

Live Tag Cloud Voting — веб-сервис для интерактивного голосования на конференциях, воркшопах, лекциях и онлайн-мероприятиях. Модератор создаёт сессию с вопросом для аудитории, зрители **свободно вводят свои слова/фразы** (без предустановленных вариантов), а результаты формируют динамическое облако тегов в реальном времени.

### Ключевые задачи

- Вовлечение аудитории через свободный ввод слов без авторизации
- Визуализация коллективного мнения через WordCloud в реальном времени
- **Объективность**: зритель НЕ видит облако тегов и чужие ответы — только своё поле ввода
- Защита от накруток: один набор голосов на пользователя через cookie-сессии
- Простой интерфейс модератора для управления сессиями
- Масштабируемость до 1000+ одновременных голосующих

### Scope MVP (v1.1)

| Включено | Вне скоупа (v2+) |
|----------|------------------|
| Создание сессии модератором | OAuth авторизация |
| Свободный ввод слов зрителями | Экспорт в PNG/PDF |
| Real-time WordCloud (polling) | Аналитика по сессиям |
| QR-код и прямая ссылка для зрителей | Кастомная цветовая схема |
| История сессий | Mobile native app |
| Защита от дублей (cookie) | PubNub / WebSocket real-time |
| Модерация слов (удаление) | PIN-защита сессий |

## 3. Пользовательские роли и потоки

### 3.1 Модератор

**Создание сессии:**
1. Переходит на `/admin`
2. Задаёт **название сессии** и **вопрос для аудитории** (опционально)
3. Нажимает «Создать сессию»
4. Получает уникальный `moderatorToken` (хранится в `localStorage`)
5. **Предустановленных тегов НЕТ** — сессия создаётся полностью пустой

**Ведение сессии** (`/admin/session/[id]`):
- Видит QR-код и ссылку для зрителей
- Видит live WordCloud с результатами
- Видит статистику: участников, голосов, уникальных слов, лидера
- Видит список всех слов с количеством голосов
- Может **удалять** нежелательные слова (модерация)
- Может открыть полноэкранный режим `/display/[id]` для проекции
- Может завершить сессию (статус → `closed`)

**Важно**: модератор НЕ может добавлять теги. Все теги создаются исключительно зрителями.

### 3.2 Зритель

**Вход в сессию:**
1. Сканирует QR-код или переходит по прямой ссылке
2. Либо вводит 6-символьный код на главной странице `/`
3. Получает анонимную cookie-сессию (`viewer_id`, HttpOnly)

**Голосование** (`/vote/[code]`):
1. Видит **только** название сессии, описание (вопрос) и пустое поле ввода
2. **НЕ видит** облако тегов, чужие ответы, статистику — для объективности
3. Вводит свои слова/фразы (до `maxVotes` штук, по умолчанию 3)
4. Видит список **только своих** отправленных слов
5. Может удалить свой ответ и ввести другой (пока сессия активна)
6. При совпадении слова (case-insensitive) с уже существующим тегом — голос прибавляется к нему
7. При уникальном слове — создаётся новый тег автоматически

**Ограничения:**
- Максимум `maxVotes` слов (по умолчанию 3)
- Максимум 50 символов на слово
- Нельзя отправить одно и то же слово дважды
- Нельзя голосовать после закрытия сессии

### 3.3 Экран трансляции (`/display/[id]`)

- Полноэкранный WordCloud для проекции на экран зала
- Обновляется каждую секунду (polling)
- Показывает: название сессии, статус, WordCloud, статистику, join-код

## 4. Архитектура

### Страницы (App Router)

| Путь | Назначение |
|------|-----------|
| `/` | Landing — создать сессию или ввести код |
| `/admin` | Панель модератора — список сессий, создание новой |
| `/admin/session/[id]` | Управление сессией, QR, WordCloud, статистика |
| `/vote/[code]` | Голосование зрителя (свободный ввод) |
| `/display/[id]` | Полноэкранный WordCloud для проекции |

### API Endpoints

Базовый путь: `/api/v1`

**Сессии:**

| Метод | Endpoint | Описание | Авторизация |
|-------|----------|----------|-------------|
| GET | `/sessions` | Список всех сессий | — |
| POST | `/sessions` | Создать сессию (без предустановленных тегов) | — |
| GET | `/sessions/[id]` | Данные сессии | — |
| PATCH | `/sessions/[id]` | Обновить статус | `x-moderator-token` |
| DELETE | `/sessions/[id]` | Удалить сессию | `x-moderator-token` |
| GET | `/sessions/join/[code]` | Найти сессию по join-коду | — |
| GET | `/sessions/[id]/results` | Агрегированные результаты | — |

**Теги:**

| Метод | Endpoint | Описание | Авторизация |
|-------|----------|----------|-------------|
| GET | `/sessions/[id]/tags` | Список тегов (для внутреннего использования) | — |
| DELETE | `/sessions/[id]/tags/[tagId]` | Удалить тег (модерация) | `x-moderator-token` |

> **POST `/sessions/[id]/tags` удалён** — теги создаются только через голосование зрителей.

**Голосование:**

| Метод | Endpoint | Описание | Авторизация |
|-------|----------|----------|-------------|
| POST | `/sessions/[id]/votes` | Отправить слово (создаёт тег если нового) | cookie `viewer_id` |
| DELETE | `/sessions/[id]/votes?tagId=X` | Удалить свой голос за слово | cookie `viewer_id` |
| GET | `/sessions/[id]/votes/me` | Мои голоса (tagId + label) | cookie `viewer_id` |

**Авторизация:**

| Метод | Endpoint | Описание |
|-------|----------|----------|
| POST | `/auth/session` | Создать анонимную сессию |
| GET | `/auth/session` | Получить текущую сессию |

### Авторизация

- **Модератор**: токен `moderatorToken` генерируется при создании сессии, хранится в `localStorage`, передаётся через заголовок `x-moderator-token`
- **Зритель**: анонимная cookie `viewer_id` (HttpOnly, Secure, SameSite=Strict, Max-Age=86400), создаётся автоматически при первом обращении

## 5. База данных

SQLite через `@libsql/client` (WASM, не требует нативной компиляции). ORM: Drizzle. Файл: `data/tagcloud.db`.

### Таблицы

**sessions**
| Поле | Тип | Описание |
|------|-----|----------|
| id | text PK | nanoid |
| title | text NOT NULL | Название сессии |
| description | text | Вопрос для аудитории |
| status | enum(active, closed) | Статус |
| join_code | text UNIQUE | 6-символьный код для входа |
| pin_hash | text | (зарезервировано для v2) |
| moderator_token | text NOT NULL | Токен модератора |
| max_votes | integer, default 3 | Макс. слов от одного зрителя |
| allow_custom | boolean | Всегда true в текущей версии |
| created_at | text | ISO timestamp |
| closed_at | text | ISO timestamp закрытия |

**tags** — создаются автоматически через голоса зрителей
| Поле | Тип | Описание |
|------|-----|----------|
| id | text PK | nanoid |
| session_id | text FK → sessions | Привязка к сессии |
| label | text NOT NULL | Текст тега |
| color | text | (зарезервировано) |
| is_custom | boolean | Всегда true в текущей версии |
| approved | boolean, default true | Для модерации |
| created_at | text | ISO timestamp |

**votes**
| Поле | Тип | Описание |
|------|-----|----------|
| id | text PK | nanoid |
| session_id | text FK → sessions | Привязка к сессии |
| viewer_id | text NOT NULL | ID зрителя из cookie |
| tag_id | text FK → tags | За какой тег проголосовал |
| created_at | text | ISO timestamp |

Уникальный индекс: `(session_id, viewer_id, tag_id)` — один зритель не может голосовать за один тег дважды.

**viewer_sessions**
| Поле | Тип | Описание |
|------|-----|----------|
| id | text PK | nanoid |
| session_id | text FK → sessions | Привязка к сессии |
| viewer_id | text NOT NULL | ID зрителя |
| vote_count | integer | Текущее количество голосов |
| last_seen | text | Последняя активность |
| created_at | text | ISO timestamp |

Уникальный индекс: `(session_id, viewer_id)`.

## 6. Компонент WordCloud

Кастомный React-компонент (`src/components/WordCloud.tsx`):
- Размер шрифта пропорционален количеству голосов (minFont..maxFont)
- Цвет — из палитры 15 цветов, выбирается через стабильный хеш от текста
- Лёгкий поворот слов для визуального разнообразия
- Glow-эффект для лидирующих слов
- Плавная анимация изменений (CSS transition 700ms)

## 7. Real-time обновления

MVP использует **polling** (клиентский `setInterval`):
- Страница модератора: каждые 1с
- Экран трансляции: каждые 1с
- Страница зрителя: **НЕ** запрашивает результаты (для объективности)

В v2 планируется переход на PubNub / WebSocket для мгновенных обновлений.

## 9. Ключевые принципы дизайна

1. **Объективность голосования** — зритель НЕ видит чужие ответы, облако, статистику. Только чистое поле ввода.
2. **Нет предустановленных тегов** — сессия создаётся пустой. Все теги — от зрителей.
3. **Case-insensitive совпадения** — "React" и "react" считаются одним и тем же тегом.
4. **Анонимность** — никакой регистрации, только cookie-сессия.
5. **Модерация** — модератор может удалять нежелательные слова, но не добавлять свои.

## 10. Метрики успеха (KPI)

| Метрика | Целевое значение |
|---------|-----------------|
| Задержка: ввод → отображение в WordCloud | < 2с (polling 1с) |

| Time-to-first-vote | < 30 секунд от входа |
| Целостность голосов (дубликаты) | < 0.1% |
